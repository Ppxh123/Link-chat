# 消息撤回三个问题修复

> Generated by Deep Copilot · Plan mode
> 2026-06-15T09:05:01.634Z

## 目标 / Goal
修复消息撤回的三个问题：1.撤回后其他人仍能看到消息 2.重新编辑后"重新编辑"链接残留 3.超2分钟撤回按钮消失

## 方案 / Approach
问题1根因是WebSocket广播由前端发起，messageId经JSON序列化精度丢失导致后端查不到消息不广播。改为后端recallMessage更新DB后直接广播。问题2用Set跟踪已重新编辑的消息ID，隐藏"重新编辑"链接。问题3始终显示撤回按钮，点击时前端检超时直接toast。

## 步骤 / Steps
1. **修复1：MessageServiceImpl.recallMessage 加 WebSocket 广播** — 在 recallMessage 更新 DB 后，直接通过 WebSocketServer.broadcastToGroup（群聊）或 getSession+sendMessage（单聊）广播撤回事件，不再依赖前端发送 WebSocket
2. **修复2：ChatWindow 加 reEditedIds Set** — 在 handleRecall 中单击重新编辑时将 messageId 加入 reEditedIds，通过 prop 传给 MessageSystem，已标记的消息只显示撤回文字不显示重新编辑链接
3. **修复3：MessageBubble 始终显示撤回按钮** — 移除 isRecallable 对菜单项的过滤，始终显示撤回。在 handleRecall 中前端先判超时：超时直接 toast 消息已超过2分钟无法撤回；未超时正常调 API
4. 编译后端 + 构建前端验证

## 涉及文件 / Affected files
- `linkchat-server/src/main/java/com/linkchat/server/service/impl/MessageServiceImpl.java` — 编辑：recallMessage 加 WebSocket 广播
- `linkchat-web/src/components/chat/ChatWindow.vue` — 编辑：handleRecall 加 reEditedIds，超时 toast
- `linkchat-web/src/components/chat/MessageBubble.vue` — 编辑：始终显示撤回按钮
- `linkchat-web/src/components/chat/MessageSystem.vue` — 编辑：接收 reEdited prop
